本目录是 DNA: Design Network Atlas 的项目设计与开发计划入口。当前原则是：先完成系统设计，再按阶段开发；每个开发阶段必须有对应测试和验收，所有阶段合并后才算完整系统。

## 当前执行边界

- 先设计完整系统，再进入阶段开发。
- 阶段之间必须可累积，不能做互相冲突的一次性原型。
- 每个阶段都有自己的测试入口、验收场景和提交边界。
- 当前仓库已有的代码只视为早期工程骨架，需要按路线图重新纳入阶段验收。
